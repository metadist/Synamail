# Synamail — Glossary

Single source of truth for terminology used across `docs/PROJECT_PLAN.md`, `docs/PROJECT_PLAN.md`, `docs/FEATURES.md`, and `docs/PROJECT_PLAN.md`. When in doubt, the **canonical term** column wins.

| User-facing / brief term | Canonical term in this project                                                       | Definition / where it lives                                                                                                                                                                                                        |
| ------------------------ | ------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| RAG key                  | **RAG file group**                                                                   | A named bucket of files in Synaplan. Endpoint: `GET /api/v1/files/groups`. Table: `BFILEGROUPS`.                                                                                                                                   |
| RULE                     | **Synapse Routing rule**                                                             | Tier-0 keyword/if-then matcher on a Synaplan routable topic. Field: `BSELECTION_RULES`. See `synaplan/docs/SYNAPSE_ROUTING.md`.                                                                                                    |
| Identification link      | **Office Dialog OAuth flow**                                                         | Sign-in via `Office.context.ui.displayDialogAsync` → `https://web.synaplan.com/addin/connect` → `POST /api/v1/apikeys`. See ARCHITECTURE §7.5.                                                                                     |
| Plugin                   | **Outlook add-in (web add-in)**                                                      | Office.js-based add-in. **Not** a COM/VSTO add-in.                                                                                                                                                                                 |
| Knowledge base           | **RAG group (typically `contact:<email>`)**                                          | Per-contact convention for grouping emails saved into Synaplan's RAG. New in Synamail; no Synaplan-side schema change.                                                                                                             |
| Outlook                  | Outlook on the Web / new Outlook for Windows / classic Outlook 2024 / Outlook on Mac | All four clients are targeted in v1. Outlook iOS / Android = phase 3.                                                                                                                                                              |
| Bridge page              | `/addin/connect`                                                                     | Vue route inside `synaplan/frontend/` that completes the OAuth handshake. Built into `ghcr.io/metadist/synaplan:latest`, deployed by `synaplan-platform`. **Only Synaplan-side change in v1.** See `docs/SYNAPLAN_INTEGRATION.md`. |
| Live Synaplan            | `web.synaplan.com`                                                                   | The hosted Synaplan instance the production add-in targets by default. Self-hosted overrides allowed via Settings.                                                                                                                 |

If a term doesn't appear in this table, **don't introduce it** — open a doc PR to add it first so every contributor sees the same vocabulary.
